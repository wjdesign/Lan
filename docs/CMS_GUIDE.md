## 三、新增一件「作品集」

### 流程

1. 後台左邊點「**作品集**」
2. 右上角點「**＋ 新增作品**」
3. 填表：

| 欄位 | 說明 |
| --- | --- |
| **標題** | 作品名稱，例：「宴客．舉杯的那一刻」 |
| **類別** | 下拉選單：白紗造型／宴客／訂婚／晚宴／媽媽妝／形象拍攝／幕後紀錄 |
| **年份** | 例 2026 |
| **地點** | 可以留空。例：「台中・宴會廳」 |
| **封面照** | **拖一張照片進去**（或點選擇檔案，從手機相簿挑） |
| **圖集照片** | 加更多張可以拖／點。第一張通常跟封面照相同 |
| **摘要** | 一兩句精華描述。會出現在卡片與分享連結預覽 |
| **工作人員** | 攝影師、禮服品牌、場地（可全空） |
| **標籤** | 例：「宴客」「暖光」「訂婚」（按 Enter 新增） |
| **真實作品？** | 真照片留勾選；如果是參考圖／示意照取消勾選 |
| **排序** | 數字越小越前面。建議 1, 2, 3...；新作品給較小數字會優先顯示 |
| **完整故事** | 段落式描述（可空白）。有編輯工具列，按粗體就粗體，**不用寫程式** |

4. 填好後點右上角「**儲存**」變成草稿
5. 確認沒問題後點「**發佈**」
6. 等 3 分鐘左右，網站會自動更新上線

### 範例（最少填這幾項就能發）

- 標題：宴客．舉杯的那一刻
- 類別：宴客造型
- 年份：2026
- 封面照：上傳一張
- 摘要：迎賓敬酒、與每一桌的家人朋友碰杯——這時的妝，要禁得起暖光與長時間的笑。
- 真實作品？：✅
- 排序：1

完成。

## 四、新增一篇「日誌」文章

### 流程

1. 左邊點「**日誌**」
2. 右上角「**＋ 新增文章**」
3. 填表：

| 欄位 | 說明 |
| --- | --- |
| **標題** | 例：「春日新娘的三種色彩語言」 |
| **摘要** | 一兩句概述，會出現在分享卡片與 Google 搜尋結果 |
| **發布日期** | 點日期選單選 |
| **封面照** | 拖一張代表照 |
| **作者** | 預設「葉小蘭」可改 |
| **分類** | 造型筆記／婚前指南／工作室日誌／造型趨勢／日誌 |
| **標籤** | 例：「春季婚禮」「色彩」 |
| **閱讀時間** | 例：「5 min」「8 min」|
| **正文** | 完整文章內容 |

4. **「正文」是 WYSIWYG（所見即所得）編輯器**：
   - 想粗體就選文字 → 點 **B**
   - 想加標題 → 點 `H2` / `H3`
   - 想加圖 → 點 📷 → 上傳
   - 想加引言 → 點 `"`
   - **不會出現任何程式碼**，跟 Google Docs 一樣
5. 寫完點「儲存」變草稿、再點「發佈」上線

## 七、多語系編輯（4 國語系：繁中 / 簡中 / 英文 / 日文）

網站是 4 國語系，所以**作品集與日誌也可以分別填每個語系的版本**。

### 工作流程

#### 寫一筆新作品 / 新文章
1. 後台新增作品 → 預設停在「繁中」分頁
2. 填好標題、摘要、故事⋯⋯所有中文欄位
3. 點上方分頁切到「英文」、「簡中」、「日文」
4. 兩個選擇：
   - **手動翻譯**：自己看著中文版填譯文
   - **自動翻譯（DeepL）**：按右上角「**Translate from zh-Hant**」（從繁中翻譯）按鈕，自動填入該語系欄位，老師再潤飾即可

### 一次性設定 DeepL（讓自動翻譯能用）

DeepL 是世界上最好的翻譯 API，**免費版每月 500,000 字**對工作室來說綽綽有餘。

#### 第一次設定（5 分鐘）

1. 註冊 **DeepL API Free**：https://www.deepl.com/pro-api （注意：要選 API 不是 Pro 訂閱版）
2. 註冊完成後在 dashboard 拿到 **API Key**（一串長字母數字）
3. 後台打開 Sveltia CMS
4. 右上角 ⋯ → **Settings** → **Advanced** → 找 **DeepL API Key** 欄位
5. 貼上 key → 儲存
6. 之後每篇文章右上角會出現「**Translate from xx**」按鈕

### 哪些欄位會自動翻譯？

| 欄位 | 是否翻譯 | 說明 |
| --- | --- | --- |
| 標題 | ✅ | 每個語系不同 |
| 摘要 | ✅ | 每個語系不同 |
| 故事 / 正文 | ✅ | 每個語系不同 |
| 地點 | ✅ | 例「嘉義」→「Chiayi」 |
| 標籤 | ✅ | 翻譯成對應語言 |
| 場地（credits） | ✅ | |
| **時間 / 日期** | ❌ 共用 | 改一處全部同步 |
| **封面照 / 圖集** | ❌ 共用 | 照片不分語系 |
| **類別** | ❌ 共用 | 是系統識別碼，不翻譯 |
| **攝影師 / 禮服品牌** | ❌ 共用 | 通常英文名 |
| **作者** | ❌ 共用 | 「葉小蘭」就是「葉小蘭」 |
| **真實作品？** | ❌ 共用 | true/false |
| **閱讀時間** | ❌ 共用 | 數字 |

### 沒翻譯的語系怎麼辦？

- 例如老師只寫了繁中，沒填英文版
- 那個作品在英文網頁上 **會自動 fallback 顯示繁中內容**（不會 404）
- 等老師有空再回 CMS 補英文版

### DeepL 支援的語系

繁中 ✓ / 簡中 ✓ / 英文 ✓ / 日文 ✓ — 全部我們的語系都支援。

## 八、常見問題

### Q：發布後多久會出現在網站上？
A：約 3–5 分鐘。如果開網站看不到，按 `Cmd+Shift+R`（Mac）/`Ctrl+Shift+R`（Win）強制重新整理；或開無痕視窗看。

### Q：上傳照片有限制嗎？
A：建議 1MB 內（手機拍的原圖可能 3-5MB，但 CMS 會自動縮小）。**沒有張數限制**。

### Q：可以從手機相簿直接挑照片嗎？
A：可以！手機點「上傳照片」會打開原生相簿挑選器。

### Q：發布到一半關掉視窗，內容會不見嗎？
A：不會。CMS 每分鐘自動儲存草稿。下次登入回到同一頁，會看到你之前打到一半的內容。

### Q：可以排程「未來才發佈」嗎？
A：可以。日誌的「發布日期」可以填未來日期。發布後，文章會等到那天才公開顯示。

### Q：手機編輯介面跟電腦不一樣？
A：對，欄位是垂直堆疊、上傳照片會呼叫原生相簿、富文本工具列會收起成浮動 bar。功能完全一樣。

### Q：發現網站有錯字或想改老照片怎辦？
A：到 CMS 找到那篇 → 修改 → 重新發佈。**修改不會弄丟舊內容**。

### Q：可以還原刪掉的內容嗎？
A：可以！所有更動都有 Git 歷史紀錄。請工程師幫忙還原（5 分鐘即可）。

## 九、如果遇到問題

- 登入不進去：清除瀏覽器的 cookie，再點「Login with GitHub」
- 上傳照片失敗：檢查網路；用 4G 重試；單張不要超過 10MB
- 找不到「發佈」按鈕：在頁面右上角，可能要先「儲存」一次才會出現
- 其他任何狀況：拍張錯誤訊息給工程師（wjdesign），他會幫你看

## 十、給工程師的設定 checklist

只有 wjdesign 需要看這一段。

### 一次性設定（約 10 分鐘）

#### A. 部署 Sveltia CMS Auth Worker（Cloudflare Workers，免費）

GitHub 不支援 PKCE OAuth，所以需要一個 token-exchange proxy。Sveltia 官方
提供現成的 worker，用 Cloudflare Workers 一鍵部署：

1. 註冊 Cloudflare 帳號（免費）：https://dash.cloudflare.com/sign-up
2. 部署 worker：
   ```bash
   git clone https://github.com/sveltia/sveltia-cms-auth.git
   cd sveltia-cms-auth
   npx wrangler login
   npx wrangler deploy
   ```
3. 取得部署完的 URL，例如 `https://sveltia-cms-auth.xxx.workers.dev`

#### B. 建立 GitHub OAuth App

> 注意：是 **OAuth App**（不是 GitHub App）

1. https://github.com/settings/developers → OAuth Apps → New OAuth App
2. Application name: `Lan Yeh Studio CMS`
3. Homepage URL: `https://wjdesign.github.io/Lan/`
4. Authorization callback URL: 上一步 worker URL + `/callback`
   例：`https://sveltia-cms-auth.xxx.workers.dev/callback`
5. Register application → 抓 **Client ID** 與 **Client Secret**

#### C. 把 Client ID/Secret 填到 Worker

```bash
cd sveltia-cms-auth
npx wrangler secret put GITHUB_CLIENT_ID
# 貼上 Client ID
npx wrangler secret put GITHUB_CLIENT_SECRET
# 貼上 Client Secret
```

#### D. 更新 `public/admin/config.yml`

把 `base_url:` 改成你的 worker URL：
```yaml
backend:
  name: github
  repo: wjdesign/Lan
  branch: master
  base_url: https://sveltia-cms-auth.xxx.workers.dev
```

commit + push 到 master 即可。

#### E. 邀請老師為 collaborator

- 在 https://github.com/wjdesign/Lan/settings/access → Add people
- 填老師的 GitHub username
- 角色給 **Write**（不需要 Admin）
- 老師收到信、接受邀請即可

### 本機開發

```bash
npx decap-server
```

然後開 `http://localhost:3000/admin/` 可以本機編輯不需要 GitHub auth。
